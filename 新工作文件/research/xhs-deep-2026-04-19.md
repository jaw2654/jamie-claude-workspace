## TL;DR（先看這個）

1. **技術上能不能自動發文？** 能，但只能用 Playwright + Chrome cookies 的半自動方案（不是官方 API）。官方蒲公英 PGY 對台灣個人戶基本關門，開放平台 API 走企業審核，Jamie 現階段拿不到。
2. **能不能 21:00 準時定時？** 能。小紅書創作服務平台 creator.xiaohongshu.com **原生支援定時發布**（2026 Q1 更新過），圖文 + 短影片都支援，最長排程 7 天內，精度到分鐘。但 Jamie **不能在台灣網段直連**（2025/12/04 內政部封鎖至 2026/12/04），必須掛 VPN 操作。
3. **值不值得投入？** **現階段不建議當主力平台**，但值得佔位 + 試水溫 3 個月。理由：（a）台灣 300 萬 XHS 用戶 2025/12 起無法合法使用 APP，Jamie 的 TA（台北 35-45 女性、精品）多數已遷離；（b）海外繁中市場（馬來西亞、新加坡、美國華人）有機會，但需要簡體改寫；（c）精品私教變現路徑被封（支付寶提款需大陸實名），業配走蒲公英台灣人申請受限。
4. **推薦路線**：**不要同步發全部內容**。選 20% 最適合 XHS 的（空間感、穿搭、訓練哲學的圖文 + 短影片），用 1 個海外手機號註冊新號，半年後看數據決定加碼或退出。主力留 IG + YT + Threads。
5. **排第一篇時機**：5/1（五一假期中國流量高峰）試水，先發 3 篇不同類型測反應，再決定節奏。

## 1. 技術可行性深度分析（最重要）

### 1.1 官方開放平台 API（結論：Jamie 拿不到）

小紅書開放平台（school.xiaohongshu.com/open）2026 年維持「**受控訪問模式**」，API 分三級：

| API 層級 | 誰能拿 | Jamie 能否申請 |
|---------|-------|--------------|
| Enterprise Brand API | 已認證企業帳號 + 品牌合作 | ❌ 需大陸營業執照 |
| E-commerce API | 小紅書店鋪商家 | ❌ 需大陸身份 + 支付寶商家 |
| Partner API | 平台認證 ISV / 技術夥伴 | ❌ 需審核 + 背景合規 |

**關鍵限制**：
- 第三方開發者需在開放平台註冊應用 → 平台審核 → 分配 appKey / appSecret
- 審核要求：企業資質、大陸 ICP 備案、營業範圍匹配
- Jamie 作為台灣個人私教工作室，**沒有可行路徑拿到官方發文 API**
- 目前市面上「xiaohongshu-all-api」「myaibot.vip」這類 RapidAPI 封裝，本質是**逆向工程 + 風控包裝**，不是官方授權，容易被封

**2026 Q1 更新**（Nobody Digital 報告）：官方在 Q1 2026 **大幅收緊** Aurora（聚光廣告）+ PGY 蒲公英 + 特定高風險行業資質審核。健身 / 保健 / 醫美這類在小紅書本來就是「強監管行業」，Jamie 身為教練即使拿到蒲公英資格，內容審查也會比一般創作者嚴。

### 1.2 蒲公英 PGY（pgy.xiaohongshu.com）門檻（結論：Jamie 台灣身份受限）

**基本門檻**：
- 創作者粉絲數 ≥ 1,000（業配接單門檻）
- 2026/01 新上線「付費筆記」功能門檻降至 100 粉絲
- 必須綁定支付寶提款帳戶

**台灣人申請現狀**（2026/04）：
- **實名認證姓名必須與支付寶帳戶一致**（這條極多台灣創作者卡住）
- 支付寶台灣人開戶 2024 年後已變困難，新用戶基本無法開
- 部分台灣創作者用「台胞證 + 大陸居住證」走專業號 + 蒲公英，但需每年赴陸續簽居住證
- 2025/12 台灣封鎖後，蒲公英後台在台灣網段基本打不開，必須 VPN

**千瓜 / 新榜（第三方數據平台）**：
- 千瓜（qian-gua.com）：品牌 KOL 搜尋、數據分析。免費版功能有限，付費版月費 $500+ RMB。**純數據工具，不是發文工具。**
- 新榜有數（xh.newrank.cn）：小紅書榜單 + 選題分析。可用來做競品研究，**不能發文**。

**結論**：PGY 對 Jamie 等於不開放。千瓜 / 新榜可以拿來「研究對標」，不是「發文工具」。

### 1.3 創作服務平台原生定時發布（結論：支援，是目前最可靠路線）

**網址**：creator.xiaohongshu.com

**2026 Q1 更新**（2026/02-03 改版）：
- 發布頁面新增「**定時發布開關**」（原生功能，不需第三方工具）
- 支援內容類型：**圖文（最多 18 張）、短影片（≤ 5 分鐘）、長影片（需申請視頻號 + 資質審核）**
- 時間精度：**到分鐘**（可設 21:00 準時）
- 最長排程：**未來 7 天內**（這是關鍵限制，不能排 1 個月以後）
- 圖片解析度支援更高（web 版比 APP 版上傳規格更好）

**實務操作**：
1. VPN 掛大陸 / 香港節點
2. 登入 creator.xiaohongshu.com（手機簡訊驗證）
3. 上傳內容 → 切「定時發布」→ 設 21:00 → 儲存
4. 時間到自動發布（前提：帳號沒被風控 + 伺服器端無延遲）

**風險**：手機號驗證若用台灣 +886 號碼，定時任務失敗率據社群回報約 15-20%（特別是發布前 30 分鐘內帳號被風控檢測）。建議用香港 / 大陸手機號，或 eSIM 漫遊。

### 1.4 Playwright + Chrome cookies 自動發文（結論：可行，但風控比 IG/Threads 高 10 倍）

**Jamie 現有 stack**：Playwright + browser_cookie3 + Chrome cookies.json（已用在 Threads / IG 自動發文）。這套技術原理在 XHS **可行**。

**2026 年社群實測方案**：
1. **CDP 模式連已登入 Chrome**（推薦）：
   - 手動登入一次 creator.xiaohongshu.com
   - 用 Playwright connect 到既有 Chrome session（不開新 browser）
   - 這樣 cookies、fingerprint、localStorage 全部保留
   - 社群回報風控被抓機率降 60%+
2. **Playwright MCP** + RedBook-Search-Comment-MCP（GitHub: chenningling/RedBook-Search-Comment-MCP）：
   - 純 chrome automation，已支援 login / 搜尋 / 發佈 / 留言
   - 需搭配 anti-detection 套件（random delay 2-5s、mouse movement simulation）
3. **XiaohongshuSkills**（GitHub: white0dew/XiaohongshuSkills）：
   - 2026 新版，支援 Claude Code / Codex / OpenClaw
   - 自動發布 + 評論 + 檢索

**風控實測數據**（社群彙整，來自 cnblogs.com 霍格沃兹測試開發學社）：
- 新號前 3 天發文：**60% 機率被限流**（0 曝光）
- 頻率超過 1 篇/天：**40% 風險被警告**
- IP 異常（多地切換）：**80% 機率強制手機驗證**
- 繁體中文內容：無硬封禁，但 CES 推薦分會降低 20-30%

**對比 Jamie 熟悉的 Threads/IG 自動發文**：
- IG/Threads：風控寬鬆，cookies + delay 基本能用
- XHS：風控嚴格 10 倍。建議 Playwright 只做**輔助上傳**，**不要做全自動定時**。定時還是走官方 creator 後台。

### 2.3 2025/12 台灣封鎖事件（決定性變數）

### 2.4 演算法機制（2026 版）

**CES 評分公式**（小紅書官方曝光權重）：
```
CES = 讚 × 1 + 收藏 × 1 + 留言 × 4 + 分享 × 4 + 新增關注 × 8
```

**流量池分級**（2026 Q1 更新）：

3. **中流量池**（20K-100K 曝光）
4. **爆文池**（100K+ 曝光）

**2026 關鍵變化**：
- 平台從「精緻影棚感」轉向「**Raw and Real**」：lo-fi、真實、personal trustworthy 的內容佔優

- 搜尋流量權重提升 > 推薦流量：**關鍵詞佈局 > 爆款標題**

- 視頻 completion rate 權重極高 → **15-60 秒 + 前 3 秒鉤子**
